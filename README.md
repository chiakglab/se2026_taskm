# タスク管理アプリ（参考例）

クライアント3（音楽フェス運営事務局）の依頼を解決する例です。
シフト管理やその他のグループも、このコードを参考にして書き換えられます。

## 解決していること

- タスクを割り当てる（担当者・締切を指定して登録）
- 進捗をリアルタイムで確認する（一覧画面でステータスが見える、ワンクリックで更新）
- 遅れに気づく（締切が近い、または過ぎているタスクは赤くハイライトされる）

## 動かし方

```
pip install -r requirements.txt
python app.py
```

ブラウザで `localhost:5555` を開く（自動で `/tasks` に移動する）。

最初はスタッフが0人なので、まず「スタッフ管理」からスタッフを追加してから、
タスクを作成すること。

他の人のPC・スマホから見たい場合は、同じWi-Fiにつないだ上で、
自分のPCのIPアドレスを調べて `http://(IPアドレス):5000` を開く。

## どこに何があるか（スライドとの対応）

| スライドの言葉 | このコードのどこ |
|---|---|
| ルート | `app.py` の `@app.route(...)`、全部で8個 |
| テンプレート | `templates/` フォルダの中の全ファイル |
| データベース | `data.db`（起動すると自動で作られる） |
| CRUD | タスクは4操作すべて対応。スタッフは作成・読取・ソフト削除（編集は省略） |
| GET / POST | `methods=["GET", "POST"]` の部分。フォーム表示はGET、保存はPOST |
| フォームからDBまでの流れ | `new_task` 関数の中、POSTのとき |
| static | `static/style.css` |

## items（在庫管理の参考例）との違い

このアプリには、テーブルが2つある（staff と tasks）。
1つのタスクが、1人のスタッフに紐づいている。
この「つながり」を表すために、`tasks` テーブルの中に `assigned_to` という列があり、
そこに `staff` の `id` が入っている。

一覧を表示するとき、2つのテーブルをJOINして、
担当者の名前を一緒に表示している（`show_tasks` 関数を参照）。

## スタッフの退職（ソフト削除）について

スタッフが辞めても、データは消さない。
`staff` テーブルに `active` という列があり、退職すると `0` になる。

- 退職したスタッフは、新しいタスクの担当者には選べなくなる
- ただし、過去にそのスタッフが担当したタスクには、そのまま名前が残る
- これを「ソフト削除」と呼ぶ。実際に行を消す「ハード削除」とは違う

理由：データを完全に消してしまうと、過去のタスク記録から
担当者の名前が分からなくなる。記録を残すために、消さずに「無効化」する。

スタッフの編集は省略している（必要なら items.html / form.html の
編集パターンをそのままコピーすれば追加できる）。

## 自分たちのアプリに変える時

- `tasks` を自分たちの対象（シフト、インシデント、問い合わせなど）に名前を変える
- `title`, `deadline`, `status` の部分を、自分たちのデータに必要な項目に変える
- クライアント1（シフト管理）の場合：ここに「同じ人・同じ時間がかぶっていないか」を
  チェックするロジックを追加する必要がある（このアプリにはまだ無い機能）

AIに「このコードを参考にして、〇〇用に書き換えて」と頼むときも、
この対応表を使って、AIが出したコードのどこがルート/テンプレート/DBか確認すること。
